## 核心概念速查

| 概念 | 说明 |
|------|------|
| `offlineUuid` | 离线端生成的唯一标识（UUID-4），服务端据此判断新增/更新 |
| `offlineDeleted` | 删除标记：0=正常，1=已删除 |
| `is_offline_data` | 数据来源：0=云端数据，1=离线生成 |
| **闭环同步** | 同步成功后立即重建 SQLite 并返回下载地址 |
| **多轮处理** | BiObject 树形结构最多 5 轮，确保父子关系正确建立 |

## API 接口速查

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v2/sync/upload` | POST | 提交离线数据同步 |
| `/api/v2/sync/attachment` | POST | 附件预上传 |
| `/api/v2/user/{id}/sqlite` | GET | 获取用户 SQLite 下载地址 |
| `/api/v2/user/{id}/sqlite` | POST | 手动触发生成用户 SQLite |
| `/api/v2/project/{id}/sqlite` | GET | 获取项目 SQLite 下载地址 |

## 故障排查流程

1. **检查同步日志**
   ```sql
   SELECT * FROM bi_sync_log ORDER BY create_time DESC LIMIT 10;
   ```

2. **检查 ID 映射**
   ```sql
   SELECT * FROM bi_id_mapping WHERE offline_uuid = 'xxx';
   ```

3. **检查实体记录**
   ```sql
   SELECT * FROM bi_building WHERE offline_uuid = 'xxx';
   ```

4. **查看应用日志**
   ```bash
   tail -f logs/sys-info.log | grep -E "SyncUpload|SqliteService"
   ```
